# Homeserver Dashboard — UI Source Of Truth

Этот файл — источник истины по дизайну, UX и механике работы интерфейса проекта.

## 1) Продуктовая цель

- Быстрый и надежный доступ к self-hosted сервисам из одной страницы.
- Минимум кликов для повседневных действий: открыть сервис, сменить категорию, поправить подпись/иконку.
- Интерфейс должен работать в локальной сети и быть устойчивым к отсутствию интернета.

## 2) Текущая дизайн-система (фактически принятая)

- Визуальный стиль: темная тема, glass-like карточки, мягкие hover-состояния.
- Основной UI-тон: low-noise, упор на читабельность и функциональность.
- Брендинг в хедере:
  - заголовок: `Homeserver dashboard`;
  - современная SVG-иконка дома;
  - бренд-блок расположен отдельно над блоком управления.
- Иерархия:
  - Уровень 1: категории;
  - Уровень 2: карточки сервисов.
- Интерактив:
  - карточка кликабельна целиком (переход на сервис),
  - кнопки действий (редактирование/удаление) не конфликтуют с кликом по карточке.

## 3) Информационная модель

`links.json` хранит:

- `rows` (в UI называются “категории”):
  - `name` — имя категории;
  - `order` — порядок категорий;
  - `collapsed` — свернута ли категория.
- `links` (карточки):
  - `url`;
  - `description`;
  - `icon`;
  - `row_id`;
  - `order` — порядок карточки внутри категории.

### Миграция

- Старый формат (только links) автоматически мигрируется при загрузке.
- Недостающие поля (`row_id`, `order`, `collapsed`) нормализуются и пересохраняются.

## 4) UX-механики (принятые решения)

### 4.1 Категории

- Есть дефолтная категория `_default`:
  - всегда сверху,
  - без имени,
  - не удаляется.
- Пользовательские категории:
  - создание,
  - переименование,
  - удаление.
- При удалении категории все карточки переносятся в `_default`.
- Создание категории находится в общей строке управления вместе с созданием карточки на широком экране.

### 4.2 Drag&Drop

- Категории:
  - можно менять порядок drag&drop;
  - сохраняется `rows[].order`.
- Карточки:
  - можно перетаскивать между категориями;
  - можно менять порядок внутри категории;
  - сохраняется полный layout карточек (`row_id` + `order`).
- Пустая категория остается валидной drop-зоной и показывает подсказку.

### 4.3 Свернуть/развернуть

- Каждая категория поддерживает `collapsed`.
- Состояние сохраняется на сервере и восстанавливается после перезагрузки.

### 4.4 Редактирование карточки

- Редактирование происходит в модальном окне с двумя смысловыми блоками:
  1) текст (название сервиса),
  2) иконка (ручная загрузка + поиск + сброс на автоопределение).

### 4.5 Адаптивность верхнего блока

- На широком экране:
  - формы создания карточки и категории находятся в одной общей горизонтальной строке управления;
  - бренд-блок расположен выше этой строки.
- На узком экране:
  - формы адаптивно переносятся;
  - элементы не должны ломать высоту строки из-за переноса текста в кнопках.

## 5) Стратегия иконок (принятая)

### 5.1 Автоопределение

Поиск кандидатов идет из:

- `<link rel="icon">`, `shortcut icon`, `apple-touch-icon`, `mask-icon`;
- `manifest` (берется наиболее подходящий icon);
- `meta` (`og:image`, `twitter:image`, `msapplication-tileimage`);
- стандартные пути (`/favicon.ico`, `/apple-touch-icon.png`, `/web/favicon.ico`, и т.д.).

### 5.2 Валидация и выбор

- Кандидаты проверяются по доступности HTTP.
- Допускаются `2xx/3xx`, а также `401/403` для локальных защищенных сервисов.

### 5.3 Локальный кэш

- Выбранные/загруженные иконки сохраняются в `icons-cache`.
- Цель: стабильное отображение иконок даже офлайн.

### 5.4 Ручная загрузка

- Валидируются:
  - расширение,
  - MIME,
  - сигнатура файла.
- Поддерживаемые форматы: `ico/png/svg/jpeg/webp/gif`.

## 6) Офлайн-устойчивость UI

- Нет зависимости от внешних CSS/JS CDN для рендера интерфейса.
- Fallback-иконка встроена как data URI.
- Поиск в Iconify требует интернет, но выбранная иконка сохраняется локально.

## 7) Нефункциональные требования

- Простая поддержка: один `index.php`, минимум внешних зависимостей.
- Понятная серверная деплой-модель (PHP + права на запись в `links.json` и `icons-cache`).
- Безопасность вывода: `htmlspecialchars` для пользовательских данных.

## 8) Технический долг / Что улучшать дальше

- Вынести CSS/JS из `index.php` в отдельные файлы.
- Добавить CSRF-токены на POST-операции.
- Добавить undo для удаления карточек/категорий.
- Добавить touch-friendly drag&drop для мобильных устройств.
- Добавить тестовый smoke-script для базовых сценариев UI.

## 9) Правила изменения UI в будущем

При любом изменении UI:

1. Сохранять инварианты:
   - дефолтная категория сверху;
   - удаление категории переносит карточки в дефолтную;
   - карточка кликабельна целиком, кнопки действий не конфликтуют;
   - порядок карточек и категорий сохраняется;
   - управление созданием карточек/категорий остается в едином control-row на desktop.
2. Проверять пустые состояния:
   - пустая категория,
   - пустой поиск,
   - отсутствие иконки.
3. Проверять офлайн-сценарий:
   - страница рендерится без интернета;
   - локальные иконки продолжают отображаться.
4. Обновлять этот файл, если изменились продуктовые/UX-решения.
